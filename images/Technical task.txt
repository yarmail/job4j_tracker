О проекте
Этот проект - консольное приложение. Пользователю отображается
меню с возможностями программы.

Программа может:
1. Добавлять заявку.
2. Заменять заявку на новою заявку по ID.
3. Удалять заявку по ID.
4. Отображать список всех заявок.
5. Производить поиск по имени заявки.

Поведение пользователя
Пользователь запускает программу и видит меню.
Программа просит ввести в консоль пункт меню для дальнейшего действия.

Например, пользователь ввел цифру 1.
Система просит пользователя ввести имя заявки.
После этого действия система сохраняет в памяти заявку
и снова отображает пункты меню.

Если пользователь выбрал пункт 6 - это будет выход из программы,
т.е. программа закрывается.

Программа имеет тривиальные действия. В ней мы будем
продолжать использовать операторы ветвления, циклы и массивы.

Но первостепенную роль в этом проекте займут объекты.
Появление любых новых требований вносить изменения практически
во все процедуры программы. С помощью ООП решим эту задачу.

Объекты
Почему проект использует такую архитектуру?
Этот проект - это стереотипная задача.
Ее уже решали тысячи программистов и на основании
их опыта можно выделить ряд решений, которые применяются
в этом конкретном проекте.

В основе любой системы лежит фундамент проекта.
В программировании - это хранилище. Вначале мы условились,
что данные в системе будут храниться в памяти.

В модуле SQL мы расширим возможности проекта.
В проекте появится возможность хранить данные в базе данных.
Расширение не будет касаться изменений существующего кода.
Это возможно за счет ООП.

Наше хранилище будет описывать класс ru.job4j.tracker.Tracker.
Этот объект будет уметь: добавлять, заменять, искать по имени,
читать все сохраненные данные.

Важный момент в системе будет занимать объект ru.job4j.tracker.Item.
Этот объект будет описывать модель данных.
Модель данных - это набор информации, описывающий объект.
Например. Паспорт - это объект, который содержит набор информации о человеке.
Паспорт, в данном контексте, является моделью данных.
О моделях данных  будет рассказано ниже в этом модуле.

Следующей задачей в нашем проекте будет отображение меню
- ru.job4j.tracker.StartUI.
Этот объект будет управлять меню. В нем будет цикл,
который будет опрашивать пользователя о выбранном пункте меню.
Если пользователь выбрал пункт "Выйти", то цикл завершается
и программа закрывается.

Действия меню будут описаны группой классов, основанных на сущности
ru.job4j.tracker.UserAction. Если от заказчика придет требование
добавить новое действие, мы сделаем еще один объект на основании
этого элемента и добавим его в систему без внесения изменений.
Это возможно благодаря ООП.

В этом проекте у нас появляется внешний ресурс
- это система ввода-вывода. В Java есть готовые реализации
для работы с вводом данных через консоль.
Но прямое использование этих классов в проекте заставит
нас ограничить проект только консольной версией.
Этого нам бы не хотелось. Так же внешний ресурс ограничивает
нас в возможности использовать тесты.
По этой причине мы введем элемент ru.job4j.tracker.Input.
Объекты этой сущности будут отвечать за ввод данных.

Здесь используется множественное определение ввода данных,
потому что в системе будут два объекта, отвечающих за ввод данных.
Один для консоли, другой для тестов.
Замена в программе этих объектов не повлечет к изменению кода.
Опять же, это возможно за счет принципов ООП.
